92. 反转链表 II

发表于 2020-03-12 分类于 LeedCode

题目描述

反转从位置 m 到 n 的链表。请使用一趟扫描完成反转。

说明

1 ≤ m ≤ n ≤ 链表长度。

示例

输入: 1->2->3->4->5->NULL, m = 2, n = 4
输出: 1->4->3->2->5->NULL

关于链表

在分析这道题前我想要吐槽一下链表.链表虽然说是非常基础的数据结构,但是其操作往往非常容易犯错,初学链表的时候我还一度把链表的指针和其内容搞混.对链表进行移动可谓是如履薄冰,一不小心链表就有可能断裂或者是访问空指针.因此做链表的题目必须小心访问空指针的问题.

问题分析

这道题的做法非常丰富,但是如果方法选的不是特别好很容易陷入头结点和尾节点的特殊处理.

如何才能将链表翻转呢？一个简单直观的方法就是将要翻转的部分的指针全部反向,然后互换头尾部的连接.

这样的处理方法最麻烦的地方在于指针反向.为了使两个节点之间的指针反向,我们必须要对前后4个节点进行访问或操作.如果翻转部分涉及到链表头部和尾部,我们必须避免访问空指针.这将为我们的代码带来不少的麻烦.

一个不错的处理方法是翻转尾差.我们可以把翻转分解为这样的一系列操作,以题目例子1->2->3->4->5->NULL m=2,n=4为例：我们先把3放到了2前面,再把4放到了3前面.通过这样的操作,我们完成了翻转.

也就是说,我们找到翻转部分的前一个节点,然后从翻转部分的第二个节点开始,不断的把后面的节点插到翻转部分的前一个节点,一直到最后一个翻转部分的节点.

每次翻转操作我们只需要一个固定的节点和两个在翻转区域内的节点.唯一的空节点访问的可能在于如果翻转部分从第一个开始,我们就没有可以翻转部分的前一个节点.但是处理方法也很简单：我们在链表的头结点前面设一个虚拟头结点即可.
